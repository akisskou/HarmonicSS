Patient Selection Module - Main Menu

This page has been created strictly for testing purposes.

Create a new request

Through this page the user can create a new Patient Selection Request. It returns the Data Access Request ID.

Manually create a new request

Through this page the user can create a new Patient Selection Request, defining all the inclusion and exclusion criteria. It returns the Data Access Request ID.

Execute a request

Through this page the user can execute a Patient Selection Request by providing the Data Access Request ID (DAR-ID). It returns the results from the Execution.

Also visit the HarmonicSS Platform for Accepting/Rejecting the pending Patient Selection Request(s): Open HarmonicSS Platform Web Page

View Execution Data

View the Results (i.e., Amount of Patients and Criteria Used) from the execution of a Patient Selection Request by Providing the Data Access Request ID (DAR-ID).

View Patient IDs

View Patient IDs results from the execution of a Patient IDs Selection Request by Providing the Data Access Request ID (DAR-ID).

Test your criteria

Through this page you can execute a particular criterion (JSON) in a specific cohort DB. This is very useful for testing purposes (e.g., ensures that the JSON can be further processed by the system).
